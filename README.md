## 🎯 問題設定

### 1. アイデアロス（Idea Loss Phenomenon）
AI時代に大量生成されるアイデアの多くは、会話が終わると消えてしまいます。人類史上最大規模のアイデア・思考の揮発現象が起きています。Cyvaiはそれらを定着させ、**“思想データ”**としてアーカイブします。

### 2. AGI の燃料不足
AIが真に渇望しているのは、整然とした正解データではありません。「なぜそう思ったのか」という人間の生々しい理由・感情・文脈のデータ群です。

---

## 🛠️ 構想：2層のアーキテクチャ

Cyvaiは、ユーザーを集めるための**「アイデアプラットフォーム（表面）」**と、そこから価値を抽出する**「思想インフラ（深層）」**の2層構造で機能するプラットフォームである。

---

## 📋 設計案（実装者へ）

> 以下は構想段階の設計アイデアです。実際の仕様・優先度・実装方法は、引き継ぐ者が自由に解釈・改善してください。

### 【表面レイヤー】 アイデア流通プラットフォーム案

* **案：摩擦ゼロの投稿フロー** — ChatGPT / Claude等との会話中に「まとめて」と依頼し、専用ボタンからワンタップで出品。ライトユーザーはGoogle等のSSOで即登録可能。AIが自動でタイトル・タグを付与。
* **案：非公開エリア（クローズド保護）** — 投稿されたアイデアはまず非公開エリアへ保管。包括的な同意と本人確認を完了したユーザーのみ閲覧・評価できる。ブロックチェーンのタイムスタンプで先行性を記録し、知的財産の権利保全を将来的に支援する。
* **案：独占交渉期間とフリー化** — 投稿から一定期間（例：90日間）は独占的な買収交渉期間とし、期限を過ぎれば自動的にフリー化（公開）される。「他社に取られる前に買う」という企業側の決断を促す。
* **案：多次元評価と AIマッチング** — 「💎 儲かりそう！(市場性)」「🧠 天才的！(革新性)」など多様な軸でレビュー。評価データが蓄積されるほど AIのマッチング精度が向上する。
* **案：ポイント経済と投稿制限** — 投稿は時間あたりの上限あり（スパム・自動生成ノイズの防止）。追加投稿はポイントを消費して解放。「人間が選び抜いた意味ある試行錯誤」だけを残す能動的な設計。
* **案：人間によるマッチング提案者（キュレーター）** — 評価で稼ぎたい人や目利きが、投稿されたアイデアを見て「この企業に提案するとよいのではないか」とマッチングを誘導・提案できる。提案が成立した際に報酬ポイントや手数料の配分を受け取れる仕組み。
* **案：評価経済圏と才能グラフ** — 投稿・評価の積み重ねが「分野別才能グラフ」として可視化され、知的な市場価値がリアルタイムに更新される。ポイントが増えるほど高単価な非公開案件へのアクセス権が拡大する。
* **案：自動契約** — 包括的な事前同意により、対人折衝なしでアイデアの売買が自動成立。マネタイズのハードルを極限まで下げる。

### 【深層レイヤー】 思想インフラとしての本質的価値案

> アイデア売買の手数料収益は二次的なものです。本当の価値は、集約された**思想データそのもの**にあります。

* **案：メタ評価と重み付け** — アイデアだけでなく「評価者」自身も相互評価され、得意分野・信頼度で重み付けされる。すべてが「評価漬け」になることで評価の解像度が自律的に最適化される。
* **案：多元性のデータ化** — 倫理・未来性・創造性など、人間の判断の多元性をそのままデータ化。Good/Badの単一軸では捉えられない複雑さを保存する。
* **案：不完全さの体系化（Crap Data Architecture）** — 迷い・誤解・無関心。削ぎ落とされがちなノイズにこそ、AIの人格形成に不可欠な学習素材が眠っている。
* **案：AI暫定評価 × 人間評価の乖離** — 投稿時にAIが即座に暫定評価を付与。後から人間がつけた評価との「ズレ」そのものが、AIの感性・判断力を磨く最も貴重な学習素材になる。

> **ビジネスモデルの真髄** > 在野のアイデアマン・クリエイターから得られる「生の人間データ・文脈データ」を、AI企業への学習基盤として提供することが本来の目的です。
